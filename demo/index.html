01 — Method

How detection works

Three stages run on every request, in sequence, before a single token reaches your model.

Stage 1

Embedding generation

The prompt is converted into a dense vector that captures intent, not just words — so paraphrased attacks look the same as their originals.

Stage 2

Semantic classification

The vector is scored against known attack clusters — role override, exfiltration, jailbreak — in under 20 milliseconds.

Stage 3

Real-time protection

Verdicts return before the request reaches your model — blocked traffic never touches inference, so nothing is billed or logged upstream.
